# Как дать ссылку одногруппникам

Это статический сайт: ему не нужен сервер, база данных или платный хостинг. Для публикации нужны только файлы:

- `index.html`
- `styles.css`
- `app.js`

PDF учебника публиковать не нужно и не стоит: словарь уже внесен в `app.js`, а сам PDF большой и может быть защищен авторским правом.

## Самый простой вариант: GitHub Pages

1. Создай новый публичный репозиторий на GitHub.
2. Загрузи в него `index.html`, `styles.css`, `app.js`, `README.md`, `.gitignore`.
3. Открой `Settings` -> `Pages`.
4. В `Build and deployment` выбери:
   - Source: `Deploy from a branch`
   - Branch: `main`
   - Folder: `/root`
5. Через минуту GitHub покажет ссылку вида:
   `https://username.github.io/repository-name/`

## Альтернатива: Cloudflare Pages

1. Создай проект в Cloudflare Pages.
2. Подключи GitHub-репозиторий или загрузи файлы вручную.
3. Build command оставь пустым.
4. Output directory: `/` или пусто, если Cloudflare предлагает root.

## Как работает прогресс

Прогресс каждого человека хранится в его браузере через `localStorage`.

Это значит:
- у каждого одногруппника будет свой прогресс;
- тебе не нужен сервер и база данных;
- если человек чистит данные браузера, прогресс сбросится;
- если он откроет сайт на другом устройстве, прогресс там будет отдельным.
